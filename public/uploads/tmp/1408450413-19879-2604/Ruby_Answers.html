RVM(Ruby Version Manager):


RVM, is a Unix like software platform designed to manage multiple installations of Ruby

on the same device.The different versions can then be switched between to enable a

developer to work on several projects with different version requirements.


First Install RVM:

i)Intially we should check for updates

Open the terminal


$sudo apt-get update

ii)If this is done ,we can start installing RVM, Ruby Version Manager. This is a program that

lets you use several versions of Ruby on one server.

$sudo apt-get install curl

To install RVM,


$\curl -L https://get.rvm.io | bash -s stable


After it is done installing, load RVM


$source ~/.rvm/scripts/rvm


In order to work, RVM has some of its own dependancies that need to be installed. To

dependencies.


iii)Install Ruby(to install specific version of ruby specift the version numbet

Ruby Blocks

You can assign a name to a block.


The code in the block is always enclosed within braces {}.

A block is always invoked from a function with the same name as that of the block.

This means that if you have a block with the name test, then you use the method test to invoke this block.

You can invoke a block by using the yield statement.



def test

   puts “This is a method”
   yield
   puts “This is a method”
   yield

    end

    test
    {
         puts “This is a block”
     }

  i)BEGIN Block
 ii)END Block

 Every Ruby source file can declare blocks of code to be run as the file is being loaded (the BEGIN blocks) and   
 after the program has finished executing (the END blocks).

MongoDB each document is saved as key=>value pair.


When Compared with Mysql in this MongoDB while creating a Collection(i.e table) we need not

to specify column names as in mysql/oracle DB's and while inserting we specify col name with

value .

to create Collection(Table in Mysql/Oracle)

db.createCollection("student")

to insert a document(record)

db.collection.insert({name:value,name:value})


In MongoDB the DB itself creates uniqueid for each document(Similar to primarykey)

And while inserting documents we need not requiredto specify the data type as in Mysql/Oracle

the MongoDB itself converts implicitly.
